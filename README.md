# SendMail

Desktop outreach app for legitimate radio pitching.

## What it does

- Loads a public college radio directory or a local CSV.
- Personalizes a pitch per station.
- Sends through your SMTP account.
- Keeps a `sent_log.csv` so you do not resend to the same address.
- Stores SMTP passwords in the OS keychain via `keyring`, not in the settings JSON.
- Supports preview, dry-run, and throttled sending.
- Uses a generated app icon for the window and packaged executable.
- Can upload oversized attachments to Google Drive, then insert the share link into the email body.

## Quick start

1. Copy `.env.example` to your shell environment or set the variables manually.
2. Start the GUI:

```powershell
python .\sendmail_app.py
```

3. Or use the CLI if you want a quick dry run:

```powershell
python .\send_pitches.py --input .\stations_sample.csv --dry-run
```

## Live send

```powershell
$env:SMTP_HOST="smtp.gmail.com"
$env:SMTP_PORT="587"
$env:SMTP_USER="your.email@gmail.com"
$env:SMTP_PASSWORD="your-app-password"
$env:FROM_EMAIL="your.email@gmail.com"
python .\send_pitches.py --input .\stations.csv --delay 2
```

## Pull stations from the site

```powershell
$env:SMTP_HOST="smtp.gmail.com"
$env:SMTP_PORT="587"
$env:SMTP_USER="your.email@gmail.com"
$env:SMTP_PASSWORD="your-app-password"
$env:FROM_EMAIL="your.email@gmail.com"
python .\send_pitches.py --source-url https://college.crockpotcartel.com/ --dry-run --max-per-run 2
```

Remove `--dry-run` after you verify the first messages.

## GUI workflow

1. Pick the source type: public URL or local CSV.
2. Enter SMTP host, port, user, password, and sender details.
3. Edit subject and body templates if needed.
4. If you plan to attach files, add them in `Files`.
5. If any file is larger than the Drive threshold, configure the `Drive` tab with your Google OAuth desktop client JSON and token cache path.
6. Click `Load stations`.
7. Preview a selected station.
8. Click `Dry run` or `Send now`.

## CSV columns

Supported optional columns:

- `station_name`
- `music_director`
- `artist_name`
- `song_title`
- `stream_url`
- `spotify_artist_url`
- `youtube_url`
- `genre_description`
- `theme_description`

## Notes

- Use this only for contacts you are allowed to email.
- Keep delays low enough to avoid triggering provider abuse protection, but not zero.
- If your provider requires OAuth instead of SMTP password login, the script needs a small auth adapter.
- For Google Drive uploads, create a Google Cloud OAuth desktop client, enable the Drive API, and store the client secrets JSON in the `Drive` tab.

## Builds

### Windows

```powershell
.\build_windows.ps1
```

This produces `dist\SendMailOutreach.exe`.

If the file is in use, the build can be emitted to another folder such as `dist_keyring\SendMailOutreach.exe` until the running process is closed.

### macOS

```bash
chmod +x ./build_macos.sh
./build_macos.sh
```

This produces `dist/SendMailOutreach.app`.

### Linux

```bash
chmod +x ./build_linux.sh
./build_linux.sh
```

This produces `dist/SendMailOutreach`.
